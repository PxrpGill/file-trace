# Индикаторы drag-and-drop и загрузки файлов в BrowserPage

Дата: 2026-07-08

## Проблема

Перетаскивание файла на рабочую область (`<main className="content">` в
`frontend/src/pages/browser/ui/BrowserPage.tsx`) работает молча: нет ни
подсветки зоны, пока файл тянут над ней, ни индикации того, что после
отпускания идёт загрузка на сервер. Пользователь не понимает, можно ли
бросить файл и загрузился ли он.

## Решение

Два независимых индикатора, оба локальны для `BrowserPage`. Backend не
меняется, новые зависимости не добавляются.

### 1. Оверлей при перетаскивании

- Состояние: `isDragOver` (useState) + счётчик входов `dragCounter`
  (useRef), потому что `dragleave` срабатывает при переходе курсора на
  дочерние элементы.
- Обработчики на `<main className="content">`:
  - `onDragEnter` — если в `e.dataTransfer.types` есть `Files`,
    инкремент счётчика и `isDragOver = true` (перетаскивание текста или
    ссылок оверлей не вызывает);
  - `onDragLeave` — декремент; при нуле `isDragOver = false`;
  - `onDrop` — сброс счётчика и `isDragOver = false` (существующая
    логика загрузки сохраняется).
- Оверлей: `position: absolute; inset: 0; pointer-events: none` поверх
  `.content` (сам `.content` получает `position: relative`), пунктирная
  рамка, полупрозрачный фон, крупный текст по центру.
- Три варианта содержимого:
  - папка выбрана и право `write` → рамка/текст цвета `--ink`:
    «Отпустите, чтобы загрузить в „{имя папки}“»;
  - папка выбрана, но только чтение → акцент `--wax`:
    «Загрузка недоступна — только чтение»;
  - папка не выбрана → акцент `--wax`: «Сначала выберите папку».

### 2. Плашка «идёт загрузка»

- В `useUploadFileMutation`
  (`frontend/src/features/file/upload-file/model/use-upload-file.ts`)
  добавляется `mutationKey: ['upload-file']`.
- В `BrowserPage` — подсчёт активных загрузок через
  `useMutationState({ filters: { mutationKey: ['upload-file'], status:
  'pending' } })` — тот же паттерн, что уже используется для
  `create-version` и `extract`.
- Пока активных загрузок > 0 — неблокирующая плашка под `content-head`:
  CSS-спиннер + «Загружается файлов: N…». Таблица остаётся доступной.
- Плашка срабатывает и при загрузке через кнопку «Загрузить файл»
  (общая мутация), не только при drag-and-drop.
- Ошибки — как сейчас: существующий модал «Не получилось».

### 3. Стили

Новые классы в `frontend/src/app/styles/styles.css` на существующих
токенах: `.drop-overlay` (+ модификатор запрета), `.upload-banner`,
`.spinner` (CSS-анимация вращения, без библиотек).

## Проверка

- `make build` (tsc -b && vite build) — проверка типов и сборка.
- Ручная проверка в браузере через `make dev`: перетаскивание в
  writable-папку, в read-only-папку, без выбранной папки; загрузка
  нескольких файлов сразу (счётчик N); загрузка через кнопку.

## Вне объёма

- Прогресс в процентах / per-file прогресс-бары.
- Drag-and-drop директорий (существующая кнопка «Загрузить структуру»
  не меняется).
- Изменения аудита и backend.
