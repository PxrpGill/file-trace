# Сворачиваемое дерево папок

Дата: 2026-07-08

## Проблема

Дерево папок (`frontend/src/widgets/folder-tree/ui/FolderTree.tsx`) всегда
полностью развёрнуто — все уровни вложенности видны сразу. Стрелка «▸»
перед именем папки декоративная и не реагирует на клик. При большом
количестве вложенных папок список становится длинным и неудобным для
навигации.

## Решение

Изменения только в `FolderTree` (компонент используется исключительно в
`BrowserPage`; в `PermissionsPage` и `MoveFileAction` используется
`flattenTree` напрямую, их это не касается). Backend и API не меняются.

### 1. Состояние развёрнутости

- Локальный `useState<Set<number>>` внутри `FolderTree` — id папок,
  чьи дети сейчас видны.
- Начальное состояние — пустой set: при открытии страницы видны только
  корневые папки, все вложенные списки свёрнуты.

### 2. Клик по строке

- Клик по `.tree-row` как и раньше вызывает `onSelect(node)`.
- Дополнительно, если у папки есть дети (`node.children.length > 0`),
  её id переключается (toggle) в set развёрнутых — один клик и
  выбирает папку, и разворачивает/сворачивает её прямых потомков.
- У папок с детьми стрелка `▸` поворачивается на 90° при раскрытии
  (CSS `transform: rotate(90deg)` + `transition`, без смены символа).
- У папок без детей стрелка не рисуется — на её месте пустой спейсер
  той же ширины (`visibility: hidden` на том же span), чтобы имена
  папок разных уровней не смещались по горизонтали.
- На кнопку добавляется `aria-expanded` (только если есть дети) для
  скринридеров.

### 3. Автораскрытие пути при выборе по URL

`BrowserPage` умеет выбирать папку по `?folder=id` (например, по
ссылке из аудита) — папка может быть глубоко вложена. Если её предки
свёрнуты по умолчанию, сама папка не будет видна в дереве.

- В `FolderTree` — `useEffect`, зависящий от `selectedId`: при каждом
  изменении ищет цепочку предков выбранного узла рекурсивным обходом
  `nodes` (новый хелпер `findAncestorIds`, локальный для компонента)
  и добавляет их id в set развёрнутых (объединяя с уже раскрытыми, не
  заменяя целиком).
- Раскрываются только предки, не сам выбранный узел — его собственные
  дети остаются свёрнуты, пока по нему не кликнут.
- Эффект отрабатывает и при обычном клике (после `onSelect` id меняется
  на тот же самый узел, у которого предки уже раскрыты — no-op), и при
  первом монтировании после загрузки дерева и подстановки `selected` из
  URL.

### 4. Стили

Правки в `frontend/src/app/styles/styles.css`: у `.tree-row .arrow`
(новый класс на span со стрелкой) — `transition: transform 0.15s`,
класс-модификатор `.tree-row .arrow.expanded` — `rotate(90deg)`.
Существующие `.tree-row`, `.tree-children`, `.lvl` не меняются.

## Проверка

- `make build` (tsc -b && vite build) — проверка типов и сборка.
- Ручная проверка в браузере через `make dev`:
  - при открытии страницы видны только корневые папки;
  - клик по папке с детьми раскрывает/сворачивает список и выбирает её;
  - клик по папке без детей просто выбирает, без визуального сдвига
    имени;
  - переход по ссылке с `?folder=<id>` на глубоко вложенную папку —
    путь до неё разворачивается автоматически, сама папка выбрана и
    видна.

## Вне объёма

- Кнопки «развернуть всё» / «свернуть всё».
- Сохранение состояния развёрнутости между перезагрузками страницы
  (localStorage и т.п.).
- Клавиатурная навигация стрелками влево/вправо для сворачивания.
